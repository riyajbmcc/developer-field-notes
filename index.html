difference between git and github

💻 Git (The Tool)

What it is: A software program that you install on your computer.
What it does: It tracks the changes you make to your code over time (like an advanced "Undo" history).
Internet needed? No. It runs completely offline on your local machine.

🌐 GitHub (The Place)

What it is: A website and cloud platform.
What it does: It hosts your Git repositories online so you can back up your code and share it with others.
Internet needed? Yes. You need to be online to upload (push) or download (pull) your code to and from GitHub.

daily loop

  • Edit: Making changes to your code files inside your text editor.

  • Stage: Selecting and preparing specific changed files to be included in your next save (git add).

  • Commit: Saving a permanent snapshot of your staged changes to your local computer's history (git commit).

  • Push: Uploading your local commits to a remote server like GitHub to back them up and share them (git push).

    Git Command

    "hello

    1. git init :Initializes a brand new, empty Git repository in your current project folder.
    2. git status: Shows the current state of your project, listing which files have been modified, staged, or are untracked.
    3. git log: Dislays a chronological history of all commits made in the project so far.
    4. git commit -m "message": Saves your staged files to your local history with a short message describing what changed.
    5. git push: Uploads all of your locally saved commits to a remote server, like GitHub.
    6. git pull: Fetches and downloads any new changes made by team members from GitHub directly into your local files.
    7. git branch: Lists all the branches in your project, or creates a new one if you add a name after it.
    8. git diff: Shows the exact line-by-line code changes you have made that haven't been staged yet.

    Things I Should Not Commit

    • .env - May contain API keys, passwords, and other sensitive information.
    • Secrets - Private credentials and confidential data should never be uploaded to GitHub.
    • node_modules - Contains installed packages and can be recreated using npm install.
    • .DS_Store - A macOS system file that is unnecessary for a project repository.
